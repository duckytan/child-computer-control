### 1.2 组件说明

| 组件 | 类型 | 职责 | 进程名 |
|------|------|------|--------|
| GuardService | Windows服务(SYSTEM) | 核心监控，进程管理，定时任务 | WinSecSvc_a1b2c3d4 |
| AgentA | 用户进程 | 守护进程A，与B互相监控 | svchost.exe (伪装) |
| AgentB | 用户进程 | 守护进程B，与A互相监控 | RuntimeBroker.exe (伪装) |
| LockOverlay | 用户进程(WPF) | 显示锁屏界面，验证密码 | LockOverlay.exe |

### 1.3 数据流

```
用户输入 → GetLastInputInfo → TimeTracker → 状态判断
                                           ↓
                                     触发锁屏/关机
                                           ↓
                                   LockOverlay显示
                                           ↓
                                    密码验证 → 解锁
```

### 2.2 关键接口

#### Named Pipe 消息接口

```csharp
public enum PipeMessageType : byte
{
    Heartbeat = 1,          // 心跳
    LockRequest = 2,        // 锁屏请求
    UnlockRequest = 3,      // 解锁请求
    RestartAgent = 4,       // 重启Agent
    ConfigChanged = 5,       // 配置变更
    GetStatus = 6,          // 获取状态
    StatusResponse = 7,     // 状态响应
    LogRecord = 8,          // 日志记录
    AddTime = 9,            // 增加时间
    PauseControl = 10,      // 暂停控制
    LockNow = 11,           // 立即锁屏
    ShutdownNow = 12         // 立即关机
}

[Serializable]
public class PipeMessage
{
    public PipeMessageType Type { get; set; }
    public string ProcessName { get; set; }
    public int ProcessId { get; set; }
    public DateTime Timestamp { get; set; }
    public string Payload { get; set; }
}
```

## 4. 进程间通信设计

### 4.1 管道通信

- **管道名称**: `ChildPCGuardPipe`
- **通信模式**: 命名管道服务器/客户端
- **序列化**: BinaryFormatter

### 4.2 消息流程

```
Agent → Pipe → Service
  │                │
  │  Heartbeat     │
  │────────────────┤
  │                │
  │  LockRequest   │
  │◄───────────────┤
  │                │
  │  StatusResponse│
  │◄───────────────┤
```

### 4.3 心跳机制

- Agent每3秒发送一次心跳
- Service超过10秒未收到心跳则判定Agent死亡
- Service立即重启对应Agent

## 5. 安全设计方案

### 5.1 进程伪装

| 原进程 | 伪装目标 | 伪装路径 |
|--------|----------|----------|
| AgentA.exe | svchost.exe | C:\Windows\System32\svchost.exe |
| AgentB.exe | RuntimeBroker.exe | C:\Windows\System32\RuntimeBroker.exe |

### 5.2 服务伪装

- 服务名: `WinSecSvc_a1b2c3d4`
- 显示名: `Windows Security Update Service`

### 5.3 密码安全

- 存储: AES-256-CBC加密
- 验证: 比较加密后的哈希值

### 5.4 三层守护

```
Layer 1: AgentA ↔ AgentB 互相监控
Layer 2: Service接收心跳，超10秒无心跳则重启
Layer 3: Service配置自动重启（sc failure）
```
