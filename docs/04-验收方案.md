## 3. 验收流程

### 3.1 验收准备

1. 确认测试环境：Windows 10/11 专业版独立虚拟机
2. 确认源代码：所有代码已提交到版本控制
3. 确认编译：dotnet build 成功，无警告和错误
4. 准备验收工具：任务管理器、计时器、记事本

### 3.2 验收执行

**阶段一：安装验收（预计30分钟）**
1. 执行 ACC-INS-01 到 ACC-INS-06
2. 记录每项测试结果
3. 如有失败，记录问题详情

**阶段二：功能验收（预计2小时）**
1. 执行 ACC-TIME-01 到 ACC-TIME-08
2. 执行 ACC-GUARD-01 到 ACC-GUARD-04
3. 执行 ACC-LOCK-01 到 ACC-LOCK-07
4. 执行 ACC-BLACK-01 到 ACC-BLACK-03
5. 执行 ACC-CFG-01 到 ACC-CFG-07
6. 执行 ACC-SEC-01 到 ACC-SEC-04

**阶段三：性能验收（预计30分钟）**
1. 打开任务管理器，观察CPU和内存占用
2. 触发锁屏并计时响应时间

**阶段四：可靠性验收（预计24小时）**
1. 启动系统，记录开始时间
2. 24小时后检查：
   - 服务是否仍在运行
   - 日志是否正常
   - 是否有崩溃记录

**阶段五：卸载验收（预计30分钟）**
1. 执行 ACC-UNINS-01 到 ACC-UNINS-04
2. 确认无残留文件和服务

### 3.3 验收记录

每项测试完成后填写：
- **测试时间**: YYYY-MM-DD HH:mm
- **测试人员**: 姓名
- **测试结果**: 通过 / 失败
- **问题描述**: 失败时的详细问题描述

### 3.4 验收报告

验收完成后生成验收报告，包含：
- 验收日期
- 验收环境
- 测试通过率（通过数/总数）
- 未通过项及原因
- 整改建议
- 验收结论
