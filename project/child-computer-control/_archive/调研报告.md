# 儿童电脑使用时间控制程序 - 调研报告

## 一、调研概述

调研目标：开发一个Windows平台下的儿童电脑使用时间控制程序
- 限制每日电脑使用时长，超时自动锁屏
- 每天晚上22:00自动关机
- 程序需要隐秘运行，不易被发现和关闭
- 具有自我保护能力，防止被任务管理器结束

---

## 二、现有解决方案分析

### 2.1 Windows系统内置家长控制

**Microsoft Family Safety**

- 微软官方的家长控制解决方案
- 通过微软家庭组(child account)实现
- 需要孩子使用微软账户登录
- 支持屏幕使用时间限制
- 云端策略强制执行，安全性高

**优点**：
- 系统原生，稳定性好
- 不易被破解
- 免费使用

**缺点**：
- 需要孩子使用微软账户(可能需要绑定邮箱)
- 设置较为复杂
- 需要云端同步

## 三、核心技术分析

### 3.1 进程隐藏与保护

要让程序不易被发现和结束，可以采用以下技术：

**方案A: Windows服务 + 进程守护**

```
用户模式 -> Windows服务(最高权限) -> 监控用户模式进程 -> 自动重启被结束的进程
```

- Windows服务可设置自动重启
- 服务进程独立于用户桌面会话
- 需要管理员权限安装

**方案B: Hook技术**

```cpp
SetWindowsHookEx(WH_GETMESSAGE, ...)
```

- 全局消息钩子监控进程状态
- 可以检测任务管理器等活动
- 阻止进程被结束

**方案C: 父子进程守护**

```
主进程 -> 创建子进程A -> 子进程A创建子进程B -> 相互监控
```

- 进程A监控进程B，进程B监控进程A
- 任何一个被结束，另一个立即重启
- 难度：需要避免死锁

### 3.2 锁屏技术

**API实现**

```cpp
// 锁定工作站
LockWorkStation();  // user32.dll

// 或者使用
ExitWindowsEx(EWX_LOGOFF | EWX_FORCE, 0);
```

**创建自定义锁屏桌面**

```cpp
CreateDesktop("LockScreen", ...);
SwitchDesktop(hDesktop);
```

可以创建一个自定义的锁屏界面，显示解锁密码。

### 3.3 定时关机

**方案1: shutdown命令**

```cmd
shutdown -s -t 3600  # 1小时后关机
shutdown -a          # 取消关机
```

**方案2: 任务计划程序**

```powershell
# 创建每天22:00执行的任务
schtasks /create /sc daily /st 22:00 /tn "AutoShutdown" /tr "shutdown -s -f"
```

**方案3: C#代码实现**

```csharp
System.Diagnostics.Process.Start("shutdown", "/s /t 0");
```

### 3.4 使用时间统计

**方案: 读取系统闲置时间**

```csharp
GetLastInputInfo(ref lastInput);
```

结合计时器定期检测用户是否活跃，累计活跃时间。

## 四、需求分析

### 4.1 功能需求

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 时间限制 | 每日最大使用时长(如3小时) | P0 |
| 超时锁屏 | 达到时长限制自动锁定屏幕 | P0 |
| 定时关机 | 每天22:00自动关机 | P0 |

- 稳定性：程序需要7x24小时运行
- 隐蔽性：孩子不能轻易发现和关闭
- 安全性：不能被轻易破解或绕过
- 低资源占用：对电脑性能影响小

### 5.2 核心模块

#### 5.2.1 Windows服务模块

- **功能**: 后台运行，定时检查，进程管理
- **技术**: .NET Windows Service / C++ Service
- **启动**: 系统启动时自动运行
- **保护**: 服务可配置"失败后自动重启"

#### 5.2.2 进程守护模块

**双进程守护模型**:

```
进程A <----相互监控----> 进程B

同时被服务监控，服务负责在两者都结束时重启
```

**实现要点**:
1. 进程A和B互相检查对方是否存活
2. 检测到对方结束立即重启
3. 同时向Windows服务发送心跳
4. 服务超过一定时间未收到心跳，主动启动进程

#### 5.2.3 时间监控模块

**使用时间统计逻辑**:

```
1. 启动计时器(每分钟执行)
2. 调用 GetLastInputInfo 获取最后输入时间
3. 如果距离现在 < 1分钟，认为用户"在使用电脑"
4. 累加使用时间
5. 达到限制调用 LockWorkStation()
```

#### 5.2.4 锁屏模块

**标准锁屏**:

```csharp
[DllImport("user32.dll")]
public static extern bool LockWorkStation();
```

**自定义锁屏界面**:
1. 创建隐藏桌面
2. 在隐藏桌面显示全屏窗口(解锁密码输入)
3. SwitchDesktop切换到该桌面
4. 验证密码后切换回用户桌面

#### 5.2.5 定时关机模块

```csharp
// 每天22:00检查
if (DateTime.Now.Hour == 22 && DateTime.Now.Minute == 0)
{
    Process.Start("shutdown", "/s /t 60 /c \"系统将在1分钟后关机\"");
}
```

或者使用任务计划程序API创建每日任务。

## 七、关键技术实现细节

### 7.1 进程互相守护

```csharp
public class ProcessGuardian
{
    private Process _monitoredProcess;
    private string _monitoredProcessName;
    private Timer _checkTimer;

    public void StartMonitoring(string processName)
    {
        _monitoredProcessName = processName;
        _checkTimer = new Timer(CheckProcess, null, 0, 5000);
    }

    private void CheckProcess(object state)
    {
        var processes = Process.GetProcessesByName(_monitoredProcessName);
        if (processes.Length == 0)
        {
            // 进程不存在，启动它
            StartProcess(_monitoredProcessName);
        }
    }
}
```

### 7.2 使用时间统计

```csharp
[DllImport("user32.dll")]
public static extern bool GetLastInputInfo(ref LASTINPUTINFO plii);

public struct LASTINPUTINFO
{
    public uint cbSize;
    public uint dwTime;
}

public class TimeTracker
{
    private TimeSpan _dailyLimit;
    private TimeSpan _usedTime = TimeSpan.Zero;
    private DateTime _sessionStart;
    private DateTime _lastResetDate;

    public void CheckAndUpdate()
    {
        LASTINPUTINFO lastInput = new LASTINPUTINFO();
        lastInput.cbSize = (uint)Marshal.SizeOf(lastInput);

        if (GetLastInputInfo(ref lastInput))
        {
            uint idleTime = ((uint)Environment.TickCount - lastInput.dwTime) / 1000;

            if (idleTime < 60) // 用户在最近1分钟内有过操作
            {
                _usedTime += TimeSpan.FromSeconds(idleTime);
            }
        }

        if (_usedTime >= _dailyLimit)
        {
            LockScreen();
        }
    }
}
```

## 八、隐私和安全注意事项

1. **本地存储**: 所有数据仅存储在本地，不上传网络
2. **密码安全**: 建议使用加盐哈希存储密码
3. **权限最小化**: 仅申请必要的系统权限
4. **应急方案**: 提供紧急解锁机制(如连续按Ctrl+Alt+N次)

## 九、总结

开发这样一个家长控制程序涉及以下核心技术：

1. **Windows服务编程** - 实现后台隐藏运行
2. **进程守护机制** - 防止被结束
3. **Win32 API** - 锁屏、关机等功能
4. **时间统计** - 准确计量电脑使用时间

**建议**:
- 优先考虑使用Windows系统内置的Microsoft Family Safety
- 如需自开发，推荐使用C# + Windows Service架构
- 核心难点在于进程保护，可考虑双进程互相守护方案
